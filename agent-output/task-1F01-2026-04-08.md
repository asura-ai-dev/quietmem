## Implementation Notes

- `Section` は Overview タブ内に閉じた軽量ローカルコンポーネントとして定義 (shared な `components/ui/Section` を作るほどの再利用見込みがまだ無い。後続フェーズで 3 箇所以上になったら昇格する)
- `ProjectCreateForm` の submit error は `projectStore.error` と別に `useState` でローカル保持している。理由: store は refresh 等で別 action を走らせると error を上書きしうるため、フォーム表示用には一度掴んだエラー文字列をローカルにスナップショットしておく方が UX 的に安定する
- slug バリデーションは Rust 側正規表現と同一の `/^[a-zA-Z0-9_-]+$/` を使用し、サーバ側 `invalid_input` に到達しないよう先取りチェック
- エラー色は指示どおり amber ではなく赤系 (#c46a6a)。sage / amber とのコントラストを崩さず、静かなトーン (quiet / calm / reliable, L01) を保つよう彩度を抑えた色を選んだ
- CSS はすべて `--bg-app`, `--bg-surface`, `--fg-muted`, `--accent-primary`, `--border-subtle`, spacing / radius トークンのみを参照し、raw 色値は赤系エラーの #c46a6a (指示どおり) と rgba 系装飾のみ
- FirstRun では `<ProjectCreateForm />` をそのまま置くため variant 分岐は実装していない (ticket の props も `onCreated?` のみ)。カード内側の余白は FirstRunRoute.module.css 側で吸収
- formatter hook が 2 度走ったが (ProjectCreateForm.tsx と OverviewTab.tsx)、読み直した結果 import 順や条件式の整形のみで意味論的変更なし
