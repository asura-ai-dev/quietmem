## `#[allow(dead_code)]` の扱いに関するメモ

warning ゼロを維持するため、この段階で dead_code を巡るコメントを整理しておく:

| 対象                                                       | 現在の allow                            | 解除予定                                           |
| ---------------------------------------------------------- | --------------------------------------- | -------------------------------------------------- |
| `error::AppError::NotFound` / `InvalidInput` (enum に包括) | 残置                                    | 1C02 で repo 層を command 経由で呼び始めたら外せる |
| `app_state::AppState::with_conn`                           | 新規付与                                | 1C02 で最初の command 実装が呼び始めたら外せる     |
| `app_state::AppState::conn` フィールド                     | 外した (with_conn 内で read されるため) | —                                                  |

いずれの allow にも理由コメントを添付済み。
