## 重要判断

- **Confirm の position**: チケットでは「formFooter の下 or 上」のいずれも可とされていたが、上に配置することで「複製ボタン (header) → 確認 UI → submit ボタン (footer)」の縦の視線誘導が成立し、操作の途中状態であることが視覚的に伝わるため上を採用
- **storeError の扱い**: 通常 submit と複製は完全に独立した state を持つため、`dupError` は `storeError` の fallback を使わず純粋に複製固有エラーのみ表示。これによりメッセージ混在を防ぐ (チケット指定通り)
- **transition プロパティ**: チケットには transition の指定がなかったが、ghost → fill 反転は無アニメーションだと唐突に感じるため 150ms ease で 3 プロパティ同期。frontend-design の方針に従い意図的に追加
- **disabled 条件**: `loading || dupLoading` 両方を含めることで、通常 submit 中も複製中も二重操作を防止 (チケット指定通り)
- **handleDuplicateCancel の loading guard**: `dupLoading` 中の cancel を無視することで、Tauri command 実行中にユーザーが UI を閉じて state が壊れる事態を回避 (チケット指定通り)
