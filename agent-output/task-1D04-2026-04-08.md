## 実装内容

### projectStore.ts

- 型: `ProjectState`
  - state: `projects: Project[]` / `selectedProjectId: string | null` / `loading: boolean` / `error: string | null`
  - actions: `refresh` / `create` / `update` / `selectProject`
- `create<ProjectState>()((set, get) => ({ ... }))` の curried 形式 (zustand v5 TS 推奨)
- 初期値: `projects=[]`, `selectedProjectId=null`, `loading=false`, `error=null`
- `refresh()`: `loading=true, error=null` → `projectService.list()` → 成功で `{projects, loading:false}`、失敗で `{loading:false, error:toErrorMessage(err)}`
- `create(input)`: `projectService.create` → `get().refresh()` して返却。失敗時は error 反映後 rethrow
- `update(input)`: 同様に `projectService.update` → `refresh()` → 返却。失敗時 rethrow
- `selectProject(id)`: `selectedProjectId` を差し替えるだけ (async なし)

### agentStore.ts

- 型: `AgentState`
  - state: `agentsByProject: Record<string, Agent[]>` / `worktreesByProject: Record<string, Worktree[]>` / `loading` / `error`
  - actions (agents): `refreshAgents` / `createAgent` / `updateAgent`
  - actions (worktrees): `refreshWorktrees` / `createWorktree` / `updateWorktree`
- 初期値: `{ agentsByProject: {}, worktreesByProject: {}, loading: false, error: null }`
- `refreshAgents(projectId)`: `agentService.listByProject(projectId)` → `agentsByProject[projectId]` を差し替え (他 project のキャッシュは保持)
- `createAgent(input)`: create 後 `refreshAgents(input.projectId)`。`input.projectId` を使うので余計な DB ラウンドトリップなし
- `updateAgent(input)`: `AgentUpdateInput` は `projectId` を持たないため、返却された Agent の `projectId` で `refreshAgents` を呼ぶ
- `refreshWorktrees(projectId)`: worktree 版
- `createWorktree(input)`: `input.projectId` で refresh
- `updateWorktree(input)`: 返却された Worktree の `projectId` で refresh (`WorktreeUpdateInput` に projectId が無いため)
- 例外はすべて error state に格納し、create/update 系は rethrow してフォーム側でも捕捉できるようにした (Phase 1E のフォーム実装が失敗時に「送信中」状態を解除できるように)

### 共通ヘルパー

- `toErrorMessage(err: unknown): string`
  - `err` が `AppErrorPayload` (code + message) なら `"{code}: {message}"` に整形
  - `Error` なら `.message`、文字列ならそのまま、それ以外は `"unknown error"`
  - 両 store に同じヘルパーを private で配置 (循環 import を避けるため小さいのでコピー)

## 重要判断

- `create → refresh` パターンを採用 (チケット Implementation Notes 通り、Phase 1 では楽観更新しない)。create/update は返却 DTO も返すが、state の一貫性は refresh 経由の一覧で保証する
- agents と worktrees を 1 store (`agentStore.ts`) に同居させた (チケット scope 明記)。理由: Overview タブでは「Agent の active_worktree_id を Worktree 一覧から選ぶ」という密結合な UI があり、両者を同時購読する構造が自然
- `updateAgent` / `updateWorktree` で refresh 時の `projectId` は返却 DTO から取る方針。`UpdateInput` に projectId を追加することは DTO 契約の変更になるため避けた
- catch した例外を `error` state に入れつつ create/update は rethrow する二段構え。フォーム側で `try/await` して送信ボタンの enable/disable を制御できるようにするため
- `selectProject` のみ非 async (I/O を伴わず state 差替えのみ)
- zustand v5 の curried `create<T>()(...)` 記法で統一 (uiStore と同じスタイル)
- パスエイリアス未設定のため import は相対パス (`../services/...`, `../types/bindings`) を使用 (task-1D03 handoff の注意点に従う)
