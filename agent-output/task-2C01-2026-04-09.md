# task-2C01 Handoff — AgentStatusBadge コンポーネント新規作成

## Spec Alignment

spec.md §4.6 (Agent Status 表示) および §7.5 / §13.3 に対応する共通バッジ基盤。4 値
(`idle` / `running` / `error` / `needs_input`) を色 + ドット + ラベルテキストの 3
要素で区別し、`role="status"` + `aria-label` で色非依存 A11y 要件を満たす。本チケ
ットは後続 task-2C02 (AgentList) / task-2C05 (LeftSidebar) で再利用される最下層
のプレゼンテーショナル部品を提供するのみで、組み込みは out of scope。

## frontend-design 出力サマリ

`frontend-design:frontend-design` スキルを呼び出し、ticket の CSS 仕様をレビュー。
判定: **グリーンライト (数値変更なし)**。根拠:

- **コンセプト**: "industrial / terminal LED" — 鋭い 1px ボーダー + 6px dot +
  0.02em tracking の mono 文字で「計器パネルの状態灯」のように読ませる
- **寸法**: md は `2px 8px` / 12px font / 総高 ~20px、sm は `1px 4px` / 11px font
  / 総高 ~17px。いずれも 28-32px 行高に収まり、14px 本文 (font-size-base) より
  一段小さくして階層 (name 主 / status 従) を明確化
- **dot = currentColor**: dot とラベルが同色で 1 塊として読める (色ずれ回避)
- **`needs_input` の極性反転** (dark text on amber-300 light bg) は意図的 —
  他 3 variant が dark-row 上の dark-bg バッジなのに対し、amber だけ光源のように
  視覚的に一等最初に目に入る = 「入力待ち」が真っ先に通知されるべきという情報階層
- **A11y**: role=status (polite) / aria-label="status: {label}" で dot+label を
  1 フレーズとして読み上げ、dot は aria-hidden="true" で装飾扱い
- **コントラスト**:
  - idle: muted on dark ≈ 4.4:1 (border で分離)
  - running: sage-50 on sage-700 ≈ 10:1
  - error: gray-50 on red-700 ≈ 8:1 (red-500 border で LED 発光感)
  - needs_input: gray-900 on amber-300 ≈ 11:1
    いずれも WCAG AA (4.5:1) を満たし、running / needs_input は AAA

変更推奨なし → ticket 記載の CSS / TSX をそのまま実装。

## 実装内容

### 新規 `src/features/agents/AgentStatusBadge.tsx`

- props: `status: string`, `size?: "sm" | "md"` (default `"md"`)
- `isAgentStatus(status)` で 4 値判定 → 範囲外 (DB 前方互換のフォールバック) は
  `variant = "idle"` / `label = 元文字列` (デバッグ可視性のため生の値を残す)
- DOM: `<span role="status" aria-label="status: {label}" data-status={variant}>`
  内に `<span class="dot" aria-hidden="true" />` + `<span class="label">{label}</span>`
- `data-status` 属性を残し CSS セレクタ / E2E テストの hook に使えるようにした
- React の import なし (state を持たない純粋 presentational)
- bindings.ts から `AgentStatus` 型だけ型 only import
- agentStatus.ts から `AGENT_STATUS_LABELS`, `isAgentStatus`
- default export
- className は `[base, variant, size].filter(Boolean).join(" ")` 形式

### 新規 `src/features/agents/AgentStatusBadge.module.css`

- `.badge` (base): inline-flex / gap var(--space-1) / padding 2px var(--space-2)
  / border-radius var(--radius-sm) / font-size var(--font-size-sm) /
  font-family var(--font-mono) / 1px solid transparent border /
  white-space nowrap / line-height 1.4
- `.size_sm`: 11px font / padding 1px var(--space-1) (LeftSidebar 用)
- `.size_md`: var(--font-size-sm) font (AgentList 用、明示)
- `.dot`: 6x6 / border-radius 50% / background currentColor / flex 0 0 auto
  (狭い flex row でも潰れない)
- `.label`: letter-spacing 0.02em (計器感を出す)
- `.badge_idle`: color var(--fg-muted) / background transparent / border-color var(--border-subtle)
- `.badge_running`: color var(--color-sage-50) / background var(--color-sage-700) /
  border-color var(--color-sage-500)
- `.badge_error`: color var(--color-gray-50) / background var(--color-danger-bg) /
  border-color var(--color-danger)
- `.badge_needs_input`: color var(--color-gray-900) / background var(--color-amber-300) /
  border-color var(--color-amber-500)

全色値は tokens.css の CSS 変数経由 (raw 色値なし)。

### 設計判断

- raw 色値ゼロ。tokens.css の semantic alias (`--fg-muted`, `--color-danger`,
  `--color-danger-bg`, `--border-subtle`) と raw パレット
  (`--color-sage-50/500/700`, `--color-gray-50/900`, `--color-amber-300/500`) のみ
- バリアント CSS クラス名はアンダースコア区切り (`badge_needs_input`) で TS の
  enum 値 `needs_input` と 1:1 対応
- サイズは sm / md の 2 種のみ (lg / xl は不要、ticket に明記)
- アニメーションなし (running blink 等は Phase 2 非対応)
- icon ライブラリなし (Phase 1 方針踏襲)
- fallback: 範囲外 status → idle スタイル + 元文字列ラベル (DB 互換 + デバッグ可視性)
