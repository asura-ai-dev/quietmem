### 3. Interim cast (`as AgentStatus`) 削除確認

Grep 結果 (`src/features/agents/` 配下):

```
AgentEditForm.tsx:275   setStatus(e.target.value as AgentStatus);
AgentCreateForm.tsx:255  setStatus(e.target.value as AgentStatus);
AgentStatusBadge.tsx:32  const variant: AgentStatus = known ? (status as AgentStatus) : "idle";
```

上記 3 件は **interim cast ではなく必要な cast** であり、削除対象ではない。理由:

- AgentEditForm:275 / AgentCreateForm:255 … `<select>` の `onChange` で `e.target.value` は DOM API 上 `string` 型に固定される。`AGENT_STATUS_VALUES` から `<option>` を生成しているため実行時値は必ず `AgentStatus` に収まるが、TypeScript 上は cast が必要 (これは TS の構造的型の限界)
- AgentStatusBadge:32 … `isAgentStatus(status)` で絞り込んだ後の narrowing cast。fallback to `"idle"` を担う

**本来削除すべき interim cast (submit body 内で `.trim() as AgentStatus` していたパターン)** は完全に消えている。確認コード:

- AgentCreateForm.tsx L119: `status,` (直接送信、cast なし)
- AgentEditForm.tsx L141: `status,` (直接送信、cast なし)
