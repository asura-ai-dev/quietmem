## 重要判断

- grid-area (`header` / `left` / `right`) の指定は各コンポーネント側の
  `.root` に置いた。`WorkspaceRoute.module.css` の placeholder セレクタは廃止。
  これにより WorkspaceRoute は壁紙 (grid 定義 + main/drawer placeholder) のみを
  責務とし、shell コンポーネントは grid-area と自身のスタイルを自己完結させる
- LeftSidebar の選択中ハイライトは「背景色全面塗り」ではなく「左辺 3px の
  アクセントバー + 暗めの背景」。ui-shell.md の visual direction (quiet / calm /
  reliable) と整合する控えめな表現
- 空状態テキスト (Projects 無し / 実行履歴無し / memory 無し) にアンバーを使った。
  エラーではなく「操作喚起」の補助色としての運用
- Project リストアイテムは button 要素で実装 (accessibility のため)。
  `aria-pressed={isSelected}` で選択状態を伝達
- RightPanel の Run ボタンは disabled 時も色味を残したかったが、opacity 0.5 で
  軽く落とす形に。Phase 1 では機能しないため過剰な主張を避ける
