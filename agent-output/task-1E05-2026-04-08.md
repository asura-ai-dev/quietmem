## Implementation Notes

- ルーティングは `react-router-dom` を導入せず、`uiStore.route` を switch するだけの軽量パターン (ui-shell.md の確定方針)
- `useEffect` の 2 つを分離した理由: `refresh()` を 1 度だけ呼ぶ初期化 effect と、`projects` 変化に応じて route を補正する effect を分けることで、refresh 完了後に確実に fallback が走る。StrictMode 下の二重マウントでも副作用は idempotent
- `FirstRunRoute` の仮フォームは `<form onSubmit>` + `preventDefault` を使って submit ハンドラを呼ぶ形にし、1F01 置換時に `<ProjectCreateForm onCreated={...} />` へ置き換えるだけで済む構造にした
- Dashboard / Settings はコンポーネント構造がほぼ同一だが、今後の差分 (Dashboard は統計、Settings は設定フォーム) に備えて別ファイルのまま残す (DRY より変更容易性を優先)
- すべての CSS でデザイントークン (`--bg-surface`, `--accent-primary`, `--color-gray-900`, spacing, radius) のみを参照
