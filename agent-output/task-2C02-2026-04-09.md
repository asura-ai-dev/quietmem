## Spec Alignment

`agent-docs/spec.md` §4.6 "Agent Status 表示" に対応。一覧 (LeftSidebar / Overview) と編集フォームの両方で同じ視覚言語を使うという要件のうち、Overview タブの AgentList 側を共通 `AgentStatusBadge` に寄せる。これにより task-2C05 の LeftSidebar 統合と同じ視覚言語 (色 + dot + ラベルテキスト / 範囲外 fallback) を保証する。

## frontend-design 出力サマリ

既存 QuietMem "quiet operator console" ルック（墨 x sage x amber / 低彩度 / mono アクセント）を維持し、本パッチは配置とスペーシングの微調整のみ。

主要決定:

- 旧 `.itemStatus` は削除。`data-status` による色分岐は AgentStatusBadge 内部で完結しているため重複不要。
- 代わりに `.itemStatusCell` という透明ラッパ（grid placement only）を新設し `<div>` で badge を包む。これにより以下を保証する:
  - `grid-column: 2; grid-row: 1` の占有を維持し、Name 行の行構造が崩れない
  - `justify-self: end` + `display: flex; justify-content: flex-end` で Name 列 (1fr) の右端に badge を寄せる。Name の長短で badge 位置が揺れない
  - `min-width: 72px` で最長ラベル `needs input` が収まる幅を先取り。agent ごとの status 違い (idle/running/error/needs_input) で列幅が揺れない
  - `align-self: center` で Name (16px) と badge (10-11px) の視覚的垂直中心を揃える
- 新モーションは追加せず、既存の行 hover ハイライトと `AgentStatusBadge` 内部のトランジションに任せる。
- アクセシビリティは `AgentStatusBadge` 自身の `role="status"` + `aria-label` に委譲。ラッパ `<div>` は装飾扱い。

### `src/features/agents/AgentList.tsx`

1. `import AgentStatusBadge from "./AgentStatusBadge";` を追加。
2. 行内の status 表示を差し替え:

   ```tsx

     {agent.status}
   </span>

   // after
   <div className={styles.itemStatusCell}>
     <AgentStatusBadge status={agent.status} size="md" />
   ```

- 旧 `.itemStatus` クラス (font-size/color/padding/border/background) と、その `[data-status="running"]` / `[data-status="idle"]` セレクタ 2 件をまとめて削除。
- 新規 `.itemStatusCell` クラスを追加（grid placement + 右寄せ + min-width のみ、視覚スタイルは一切持たない）。
