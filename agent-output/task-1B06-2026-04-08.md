## Important Decisions

- DTO の `agent_id` / `base_branch` / `status` すべてを `Option<String>` + `#[serde(default)]` にした。これは agent repo (1B05) でも同じパターンで、フロント TS 型の optional (`agentId?: string | null` 等) と一致する
- `create` のデフォルト適用は「空文字列も既定値に置換」する挙動 (`.filter(|s| !s.is_empty())` + `unwrap_or_else`) で統一。これも 1B05 と揃えた
- update の `branch_name` / `path` に空文字を渡された場合 InvalidInput を返す。`base_branch` と `status` は既存値を上書きできるためチェック対象外 (DB の DEFAULT 制約にも引っかからない)
- テストは `:memory:` DB + `run_pending` で実施し、1 テスト 1 接続 (`setup_db` ヘルパ) とした。db-schema.md §テスト方針に準拠
