  - `create(conn, input) -> AppResult<Project>`
    - `name` / `slug` / `root_path` を `trim` してから空チェック → `InvalidInput`
    - `slug` を `chars().all(|c| c.is_ascii_alphanumeric() || c == '-' || c == '_')` で検証 → 違反なら `InvalidInput`
    - `Uuid::now_v7().to_string()` で id 採番
    - `chrono::Utc::now().to_rfc3339()` を `created_at` / `updated_at` に入れる
    - INSERT 失敗時に `is_unique_violation` (`SqliteFailure` + `ErrorCode::ConstraintViolation`) を判定し、該当すれば `InvalidInput` にマップ
    - それ以外の rusqlite エラーは `AppError::from` 経由 (`#[from]`) で `AppError::Db`
  - `list(conn) -> AppResult<Vec<Project>>`
    - `SELECT ... FROM projects ORDER BY updated_at DESC`
    - `row_to_project` ヘルパで列名アクセス (`row.get("id")` 等) → 列順序変更に強い
  - `update(conn, input) -> AppResult<Project>`
    - `find_by_id` で既存行を取得 (`OptionalExtension::optional`) → 無ければ `NotFound`
    - `None` のフィールドは既存値を保持するマージ実装
    - `Some(v)` の値も trim + 空チェック + slug 検証を通す
    - `updated_at` だけ現在時刻に更新 (`created_at` は保持)
    - UNIQUE 違反を `InvalidInput` にマップ
    - 更新後の `Project` を構造体で組み立てて返す (RETURNING を使わず、SELECT の往復も避ける)
  - `#[cfg(test)] mod tests` に 7 テスト追加
    - `setup_db()` ヘルパ (`open_in_memory` + `run_pending`) を定義
    - `sample_input(slug)` で重複テストしやすいテンプレートを作成
