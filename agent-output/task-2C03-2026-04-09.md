# task-2C03 handoff - AgentCreateForm status select 化

## Spec Alignment

spec.md §4.3 (Agent 作成) と §7.2 の受け入れ条件 "status を 4 値以外に設定する手段が UI 上に存在しない (select で値を制限する)" に対応する。自由文字列 input を `AGENT_STATUS_VALUES` 駆動の `<select>` に置換することで、物理的に 4 値以外を入力できなくする。同時に Phase 2B Eval で known_gap になっていた interim cast (`as AgentStatus`) を削除し、state 自体を `AgentStatus` 型に格上げする。

## frontend-design 出力サマリ

**方針**: `.select` は **既存 `.input` と視覚的に 1 ピクセルも異ならない** ベースラインにロックする。Phase 1 の "dark canvas + sage hairline + uppercase micro-labels" language を破らないことが最優先 (目立たないことが正解)。

**採用した具体値**:

- padding: `10px var(--space-3)` (`.input` と完全一致)
- background: `var(--bg-app)` / border: `1px solid var(--border-subtle)` / radius: `var(--radius-sm)` / color: `var(--fg-primary)` / font-size: `var(--font-size-base)` (全て `.input` と一致)
- transition: `border-color 120ms ease, background 120ms ease` (`.input` と一致)
- hover: `border-color: var(--color-gray-400)` (`.input` と一致 / チケット指示には明示されていないが、並列した際の整合性のため追加)
- focus: `outline: none; border-color: var(--accent-primary)` (`.input` と一致。outline ring を足すと `.input` と違って見えるため不採用)
- `appearance: auto`: OS ネイティブの dropdown caret を維持。カスタム caret や custom popover は scope 外 (Tauri の webview 差分回避)

**意図的に見送った項目**:

- custom dropdown panel: native select の開いた時の popover は OS 制御のため styled できない。ここを custom select に置き換えるのは task-2C03 の scope を超える。本チケットでは「閉じた状態だけ `.input` と視覚同一」を目標とする
- status 別カラーリング (idle gray / running sage / error red / needs_input amber): spec.md §4.6 で Status 表示の視覚言語として定義済だが、これは AgentList / LeftSidebar の バッジ側 (task-2C05 以降) の責務。作成フォームの select 内で色分けを先行実装すると視覚言語が分散する

## 変更内容

### `src/features/agents/AgentCreateForm.tsx`

1. import に `AGENT_STATUS_LABELS`, `AGENT_STATUS_VALUES` を追加 (`AgentStatus` は既に import 済)
2. ローカル定数 `DEFAULT_STATUS = "idle"` を削除 (使用箇所なし)
3. state の型を `useState<AgentStatus>("idle")` に変更 (従来は `useState("")`)
4. **interim cast の削除**: submit ロジックから `trimmedStatus` 変数と `(trimmedStatus.length > 0 ? trimmedStatus : DEFAULT_STATUS) as AgentStatus` 全体を除去し、`status,` を直接渡す形に変更。state 自体が `AgentStatus` なのでキャスト不要 (Phase 2B Eval の known_gap 解消)
5. resetForm で `setStatus("")` → `setStatus("idle")`
6. status 入力部を `<input type="text">` から `<select>` に置換。option は `AGENT_STATUS_VALUES.map((v) => ...)` で生成し、表示は `AGENT_STATUS_LABELS[v]` (→ `needs_input` は "needs input" と正規化表示)
7. status hint テキストを「省略時 "idle"」→「作成直後の状態 (デフォルト: idle)」に更新
8. `onChange` 内で `e.target.value as AgentStatus` にキャスト。これは React の `HTMLSelectElement.value` が string 型のため構造的に必要な狭域キャストで、interim cast とは性質が異なる (option が全て `AgentStatus` のリテラル文字列から生成されているため型安全)

### `src/features/agents/AgentCreateForm.module.css`

- `.select` クラスを追加 (既存 `.input` と完全に同じ padding / background / border / radius / font-size / transition / hover / focus 挙動)
- `.inputInvalid` の直後に追加

### 触っていないもの (Out of scope 尊重)

- `name` / `role` / `adapterType` / `promptPath` / `configPath` フィールドは一切変更なし
- `validate()` は status を含めない (select が有効値のみ返すため追加チェック不要)
- `submitError` バナーの挙動はそのまま
- `onCreated(agent)` callback はそのまま
- AgentEditForm (task-2C04 の scope)
- LeftSidebar Agents 統合 (task-2C05 の scope)
