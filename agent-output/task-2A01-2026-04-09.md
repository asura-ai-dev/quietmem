## 実装した内容のサマリ

`src-tauri/src/domain/agent.rs` に以下を追加:

1. **`pub const AGENT_STATUS_VALUES: &[&str]`**
   - `["idle", "running", "error", "needs_input"]` の 4 値固定配列
   - 順序は UI の select option 順を兼ねる (idle → running → error → needs_input)
   - `enum` 化はしない (`Agent.status: String` を変えないため、QTM-009 まで保留)
2. **`pub fn validate_agent_status(status: &str) -> AppResult<()>`**
   - ホワイトリストに含まれれば `Ok(())`
   - 範囲外は `AppError::InvalidInput("status must be one of idle|running|error|needs_input (got: \"...\")")` を返す
   - ケースセンシティブ (`"Idle"` / `"RUNNING"` は reject)
   - 空文字も範囲外として reject
   - エラー文言は英語固定 (i18n は QTM-009)
3. **`#[cfg(test)] mod tests`** に 5 ケース:
   - `validate_agent_status_accepts_all_four_values` — 4 値全て Ok (`AGENT_STATUS_VALUES` イテレート + 直接記述の二重防御)
   - `validate_agent_status_rejects_unknown_value` — `"unknown"` → `InvalidInput`、メッセージに `"status must be one of"` が含まれる
   - `validate_agent_status_rejects_empty_string` — `""` → `InvalidInput`
   - `validate_agent_status_is_case_sensitive` — `"Idle"` / `"RUNNING"` / `"Needs_Input"` が全て reject
   - `agent_status_values_contains_exactly_four_values` — `len == 4` + 各値存在チェック

既存 `Agent` / `AgentCreateInput` / `AgentUpdateInput` 構造体は変更していない。
`use crate::error::{AppError, AppResult};` をファイル冒頭に追加。
