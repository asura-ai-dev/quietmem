## Known Gaps

- Worktree の編集 UI (name / path / status の update) は本チケットのスコープ外 (`agentStore.updateWorktree` は store 側に存在するが UI は未接続)。後続チケットで必要になった段階で検討。
- `agentId` 入力 UI は意図的に省いている。Worktree と Agent の紐付けは Agent 側 (task-1F04 の AgentEditForm) の `activeWorktreeId` 選択で行う設計 (1F04 で統合される予定)。
- `status` フィールドは自由入力 (text input)。enum 化するかは後続 UX 改善の範疇。
- 既存 AgentList の `itemWorktree` 表示 (`worktree: <id>`) は id 文字列表示のまま。id → branchName の解決は 1F04 で AgentEditForm を実装する際に必要になる可能性があるため、この handoff に記録しておく。

## Notes for Next Agent

- `agentStore.worktreesByProject[projectId]` は create/update 成功後に store 側で自動 refresh されるため、1F04 の AgentEditForm から `activeWorktreeId` を更新する際は手動で refreshWorktrees を呼ぶ必要はない。
- Worktree の status は自由文字列 (Rust 側 DTO も `String`)。現状のバリデーションは空文字のみ禁止。フロント側で enum 化する場合は `WorktreeCreateForm` の `status` input を `<select>` に差し替える単純変更で済む。
- WorktreeList / WorktreeCreateForm は AgentList / AgentCreateForm と同じデザイン言語 (sage アクセントバー, amber ドット空状態, 11px uppercase label, #c46a6a エラーカラー) を踏襲しており、既存の tokens.css 以外には新規 CSS 変数を追加していない。
- WorktreeList は `<li>` を button 化していない (選択操作が不要なため)。AgentList は選択を持つので button、Worktree は静的表示という非対称は意図的。1F04 以降で Worktree 側にも選択操作が必要になった場合は AgentList のパターンに揃えればよい。
