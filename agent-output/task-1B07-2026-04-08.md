## Completed

- `src-tauri/src/app_state.rs` を刷新
  - `AppState { paths: AppPaths, conn: Mutex<Connection> }` に拡張
  - `impl AppState { pub fn initialize() -> AppResult<Self> }` を新設
    - `AppPaths::resolve()` → `paths.ensure_base()` → `db::connection::open(&paths.db_file())` → `db::migration::run_pending(&mut conn)` → `Ok(Self { paths, conn: Mutex::new(conn) })`
  - 1A04 で `paths` フィールドに付けていた `#[allow(dead_code)]` を削除 (runtime で参照されるため)
  - `conn` フィールドには Phase 1B 後半で commands 層が参照するまで暫定で `#[allow(dead_code)]` を付与 (理由コメント付き)
- `src-tauri/src/lib.rs` を刷新
  - `run()` 内で直接 `AppPaths::resolve()` / `ensure_base()` を呼んでいたコードを `AppState::initialize()` 呼び出しに置き換え
  - 失敗時はチケット指示に従い `eprintln!("[quietmem] failed to initialize app state: {e}");` + `return` (1A04 の `std::process::exit(1)` ではなく)
  - 成功時に `println!("quietmem: initialized db at {:?}", state.paths.db_file());` を出して手動確認を容易にする (チケット Implementation Notes に準拠)
  - `use paths::AppPaths;` は不要になったため削除
  - `tauri::Builder::default().setup(...).manage(state).run(...)` は従来どおり
- `src-tauri/src/paths.rs` から 1A04 付与の impl ブロック単位 `#[allow(dead_code)]` を削除
  - `resolve` / `ensure_base` / `db_dir` / `db_file` / `projects_root` は runtime で参照されるため allow 不要
  - 未使用メソッド (`project_dir` / `agent_dir` / `agent_prompt_dir` / `agent_config_dir` / `agent_raw_dir` / `run_log_dir` / `snapshots_dir`) にはそれぞれ per-method `#[allow(dead_code)]` + 将来用途コメントを付与
  - `with_root` は既存の allow を維持 (テスト専用)

## Known Gaps

- `AppState::initialize()` 自体の単体テストは追加していない。理由: 実行すると OS の実 AppData ディレクトリ (`~/Library/Application Support/QuietMem/`) に副作用を残すため、テスト環境としてクリーンではない。構成要素 (`AppPaths::resolve` / `ensure_base` / `connection::open` / `migration::run_pending`) はそれぞれ単体テスト済みであり、統合検証は `cargo tauri dev` 起動時の手動確認に委ねる
- `AppState.conn` は現時点で read されず `#[allow(dead_code)]` を付与。後続チケット (Tauri command 実装) で解除される想定
- 起動時の初期化失敗ダイアログは未実装 (チケット指示どおり `eprintln!` + `return` に留める)
