# task-2C04: AgentEditForm の status を select 化

## Spec Alignment

spec.md §4.4 (Agent 編集: status を select で 4 値から選択) および §7.3 / §7.5

存在しない) に対応する。これにより task-2B01 Eval known_gap (interim cast) を解消する。

## Frontend Design 判定サマリ

`frontend-design:frontend-design` スキルを呼び出し、視覚一貫性チェックを実施。

### 判定結果: **CSS は 1 行も変更しない**

理由:

1. `AgentEditForm.module.css` には既に `.select` クラスが完全な形で存在する
   (Active Worktree 用として task-1F04 以来)。
   - `.input, .select` 共通基底: `padding: 10px var(--space-3)`, `background: var(--bg-app)`,
     `border: 1px solid var(--border-subtle)`, `border-radius: var(--radius-sm)` で
     旧 `<input type="text">` と寸法完全一致。
   - hover / focus / disabled の 4 状態定義済み (sage focus ring,
     gray-400 hover, opacity 0.45 disabled)。
   - `.select` 固有の chevron は 2 本の `linear-gradient` で sage 色で描画、
     `padding-right: var(--space-8)` で chevron 分を確保。
2. status `<select>` を Active Worktree select と同じ `.select` クラスで
   共有することで、編集フォーム内の 2 つの select が同一視覚言語に統一される。
3. AgentCreateForm (task-2C03) も同じ「sage chevron + bg-app + sage focus border」
   の設計言語に立っているため、別ファイル間でも自動的に視覚一貫性を維持。
4. 実装者への注意点: `.inputInvalid` は status では使わない (validate から
   status チェックが削除されるので invalid 状態が発生しない)。
   `aria-invalid` / `aria-describedby` / error span も status では不要にする。

## 変更内容

### `src/features/agents/AgentEditForm.tsx`

1. **import 拡張**: `./agentStatus` から `AGENT_STATUS_LABELS`,
   `AGENT_STATUS_VALUES`, `isAgentStatus` を追加 import (AgentStatus 型は既存)。
2. **doc comment 更新** (L30): バリデーション方針の status 行を
   「`<select>` で有効値を物理的に制限 (range 外不可。task-2C04)」に更新。
3. **FieldErrors interface**: `status?: string` フィールドを削除 (L48-52)。
4. **useState 初期化**: DB 互換性フォールバック (`isAgentStatus(agent.status) ? agent.status : "idle"`)
   経由で `useState<AgentStatus>` 化 (L69-73)。
5. **useEffect reset**: Agent 切替時の `setStatus(agent.status)` を
   `setStatus(isAgentStatus(agent.status) ? agent.status : "idle")` に変更 (L87)。
6. **validate()**: `status.trim().length === 0` チェック削除。コメントで理由を明記 (L113)。
7. **status 入力部の置換** (L266-285):
   - `<input type="text">` + inputInvalid class + aria-invalid + error span
     をまるごと削除。
   - `<select id={statusId} className={styles.select} value={status}
onChange={...}>` + `AGENT_STATUS_VALUES.map` で option 生成、
     表示は `AGENT_STATUS_LABELS[v]` ("needs_input" → "needs input") を使用。
   - Active Worktree select と同じ `.select` クラスを共有。
8. **interim cast 削除** (L140-141):
   - 旧: `// interim cast: 自由文字列 input の暫定対応 (本格修正は task-2C03 で select 化)`
     `status: status.trim() as AgentStatus,`
   - 新: `// <select> で AgentStatus の有効値が固定されているため .trim や cast は不要`
     `status,`
   - **これが Phase 2B Eval known_gap (L135 付近の interim cast) の解消に相当する**。

### `src/features/agents/AgentEditForm.module.css`

**変更なし**。frontend-design 判定通り、既存の `.select` クラスを
status select にも共有する (追加宣言不要)。
