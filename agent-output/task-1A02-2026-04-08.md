## Key Decisions

### 1. tsconfig 構成 (solution-style 3 ファイル)

- `tsconfig.json` は `files: []` + `references` のみのソリューション設定
- `tsconfig.app.json` に `src/` 用の strict + JSX + bundler 設定
- `tsconfig.node.json` に `vite.config.ts` 用の ES 設定
- 両 child を `composite: true` + `tsBuildInfoFile` 付きに。`tsc -b` と `tsc --noEmit` の両方で動作する構成を採用
- チケットには `tsconfig.json` と `tsconfig.node.json` のみが列挙されていたが、`tsc --noEmit` がソリューション設定でも成立するよう `tsconfig.app.json` を追加。ticket の Spec (strict / noUnusedLocals / noUnusedParameters / noFallthroughCasesInSwitch / moduleResolution bundler / jsx react-jsx) は `tsconfig.app.json` に反映済み
- この 3 分割は Vite の公式 react-ts template と同じ構成なので、後続チケットで拡張しても破壊的にならない

### 2. packageManager

- `package.json` の `packageManager = "pnpm@9.15.9"` (pnpm@9 系で固定)
- corepack 経由で pnpm 9.15.9 がそのまま引き込まれることを確認済み
- Phase 1 の残りチケットも pnpm 9 系で揃う想定

### 3. dependencies のバージョン

- React 18.3.1 (18 系の最新安定)、@tauri-apps/api 2.x (Tauri v2 に合わせる)
- Vite 5.4.x、TypeScript 5.6.x、@vitejs/plugin-react 4.x
- tech-stack.md の "React 18 以上 / TypeScript 5 以上 / Vite 5 以上" を満たす最小バージョン帯

### 4. StrictMode の採用

- `src/main.tsx` では `createRoot(...).render(<App />)` を素のまま書くのではなく `<StrictMode>` で包んだ。Vite 公式 template に倣った形で、ticket の記述 (`createRoot(document.getElementById("root")!).render(<App />)`) を壊さず重ねている
- 機能は同じで、後続チケットの開発で React 由来の副作用検出に役立つため
