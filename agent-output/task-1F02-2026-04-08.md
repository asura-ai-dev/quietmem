## Implementation Notes

- AgentList / AgentCreateForm の CSS は ProjectList / ProjectCreateForm を 1:1 でコピーし、必要な差分 (agent 固有の status badge, 3 行グリッド, 省略可能フィールドの "optional" ラベル) のみ追加。デザイン言語 (sage accent bar + uppercase label + amber dot empty state + #c46a6a エラー) は完全に揃えている
- `onSelect` は toggle 挙動 (選択中を再クリックで null に戻す)。1F04 の AgentEditForm に渡す前提で、AgentList 側が「選択解除」の責務を持てるようにしている
- `AgentCreateForm` の status field は default "idle" を Rust 側に送る (空文字でも "idle" を送る)。理由: Rust 側の AgentCreateInput.status は Option で、None が来ると DB 層で何を入れるかが不透明なため、フォームレベルで常に明示値を送る方が保守的
- `adapterType` の初期値は `"cli"` を `useState("cli")` で与えている。required なので空にはならないが、ユーザーが一度クリアして再入力しないまま submit すると先バリデーションでブロックされる (意図通り)
- promptPath / configPath は `string | null` 型で null を明示的に送る (空文字を送らない)。Rust 側が Option<String> で両方扱えるはずだが、null の方が「未設定」を強く表現できる
- `useEffect(() => { void refreshAgents(projectId); }, [projectId, refreshAgents])` で projectId 変更時に自動再取得。void は Promise を捨てる意思表示 (no-floating-promises 対策)
- OverviewTab の `selectedAgentId` は Overview 内に閉じた local state。`agentStore` に入れる選択もあったが、「選択中の概念は UI 層」かつ「1F04 でしか参照しない」ため、ticket の指示通り OverviewTab レベルに留めた
- `useAgentStore.loading` は agents / worktrees 両方のアクションで共有されているため、将来 Worktree フォームが同じ OverviewTab に載ったときに AgentCreateForm の submit ボタンが意図せず disabled になる可能性がある。Phase 1 の範囲では許容し、1F03 / 1F04 実装時に必要なら store 側で `agentLoading` / `worktreeLoading` に分離する
- エラー色 `#c46a6a` は ProjectCreateForm と共通。将来 tokens.css に `--error-fg` として昇格する候補だが、1F02 の scope 外
- Section コンポーネントは OverviewTab.tsx 内に閉じた軽量ローカル定義のまま (1F01 の判断を継承)。shared ui に昇格するかは 1F03 でも同じパターンが使われた後に検討する
