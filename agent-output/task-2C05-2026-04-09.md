## Frontend Design サマリ

**Design direction: "Restrained Field Notebook"** — 既に確立されている quietmem LeftSidebar の sage-green minimal 美学 (archival index / silent muted palette) にシームレスに溶け込ませる方針。新規の美学を発明せず、既存リズムに Agent 行を**対比的に配置**することで 2 セクションの性格を分ける。

### 差別化ポイント

1. **Projects は 2 行 vertical (name + slug)** ↔ **Agents は 1 行 horizontal (name + 右端 badge)** というタイポグラフィ対比でセクションの性格を区別
2. 選択中ハイライトは Projects の `.itemButtonSelected` と**同じ sage 左 2px border** パターンを踏襲 → ユーザーは既に学習済みの視覚言語を再利用
3. 選択中の `color` は `var(--accent-primary)` に切替 (Projects は `--fg-primary` のまま)。Agents は「操作対象」の意味合いが強いためより明示化
4. Badge は `flex: 0 0 auto` で右端固定、name は `flex: 1 1 auto` + `min-width: 0` + `text-overflow: ellipsis` で優雅に切り詰め → 長い Agent 名でも badge 位置が暴れない
5. raw 色値ゼロ。すべて design token (`--accent-primary`, `--bg-surface`, `--fg-primary`, `--space-*`, `--radius-sm`, `--font-size-sm`) 経由

## ZUSTAND 回帰防止パターン (EMPTY_AGENTS)

```tsx
// component 外の module 定数
const EMPTY_AGENTS: readonly Agent[] = [];

function LeftSidebar() {
  const agents = useAgentStore((state) =>
    selectedProjectId
      ? (state.agentsByProject[selectedProjectId] ?? EMPTY_AGENTS)
      : EMPTY_AGENTS,
  );
  // ...
}
```

なぜ必須か: selector 内で `?? []` や `?? EMPTY` (関数内ローカル定数) を使うと、毎 render で新しい配列参照が返り、zustand が state 変化と誤認して再 render → 無限ループ → `Maximum update depth exceeded` エラー → Workspace がブランクになる。Phase 1F task-1F06 で同じ回帰が発生した経緯があるため、**必ずコンポーネント外に `const EMPTY_AGENTS: readonly Agent[] = [];` を宣言して `?? EMPTY_AGENTS` で参照する**。
