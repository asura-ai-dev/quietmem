## Important Decisions

- `AppError` のバリアント順序は `tauri-commands.md` のサンプルに合わせたが、`Db` だけ Phase 1B に延期。`code` 文字列は `not_found` / `invalid_input` / `io_error` / `internal` で確定。
- `ensure_base` は `db/` と `projects/` のみ作成 (ticket notes 通り)。agent / run 配下は必要時に呼び出し側が作る設計。
- テストは `std::env::temp_dir()` + `uuid::Uuid::now_v7()` で衝突回避。外部 crate (tempfile 等) を追加せず、既存依存のみで成立させた。
- `dead_code` 警告はモジュール単位で `#[allow(dead_code)]` をつけ、Phase 1B で外す旨をコメントで明示。警告ゼロ状態を維持することで後続チケットのビルド出力を読みやすく保つ。
- `run()` 内の初期化失敗は `eprintln!` + `process::exit(1)` にした。チケット指示通り (`eprintln!` でログ出力)。Phase 1 の非機能要件 §6「Tauri バックエンドの初期化失敗時に最低限のログを出すこと」を満たす最小実装。
