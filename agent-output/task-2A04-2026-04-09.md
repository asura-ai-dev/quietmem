# task-2A04: repo::agent::duplicate 実装 + 単体テスト - handoff

## Spec Alignment

本チケットは spec.md §4.5「Agent 複製」§6「agent_duplicate command」§7.4「受け入れ条件 (複製)」§13.4「API 評価観点」に対応する。Phase 2 の「元 Agent の設定を引き継ぎつつ status=idle / active_worktree_id=null / memory 非引継ぎ」で新 Agent を作成する挙動を、repo 層の純粋関数として実装・テストで担保する位置付け。

## サマリ

- `src-tauri/src/db/repo/agent.rs` に `pub fn duplicate(conn, input: AgentDuplicateInput) -> AppResult<Agent>` を追加
  - `find_by_id` で source を取得 (None → `AppError::NotFound`)
  - 新 name: `input.name` が `Some(raw)` なら `raw.trim()` (空なら `InvalidInput`)、`None` なら `format!("{} (copy)", source.name)`
  - 新 status: 強制 `"idle"` を `validate_agent_status` で保険検証
  - 新 id: `Uuid::now_v7().to_string()` / now: `Utc::now().to_rfc3339()`
  - `project_id` / `role` / `adapter_type` / `prompt_path` / `config_path` は source からコピー
  - `active_worktree_id` は強制 `None`
  - INSERT は既存 `create` と同一カラム順
  - `raw_memory_entries` / `curated_memories` には **一切触れない** (= memory 非引継ぎの実装)
- ファイル先頭 `use crate::domain::agent::{...}` に `AgentDuplicateInput` を追加
- `#[cfg(test)] mod tests` に単体テスト 8 件を追加
  1. `duplicate_returns_new_agent_with_copied_fields`
  2. `duplicate_default_name_appends_copy_suffix`
  3. `duplicate_with_custom_name_uses_input` (そのまま入力 + trim の 2 ケース)
  4. `duplicate_with_empty_custom_name_returns_invalid_input`
  5. `duplicate_with_missing_source_returns_not_found`
  6. `duplicate_does_not_touch_memory_tables` (raw/curated の COUNT 不変を検証)
  7. `duplicate_starts_with_idle_status_even_if_source_was_running`
  8. `duplicate_starts_with_null_active_worktree_even_if_source_had_one`
- 既存の `create` / `update` / `list_by_project` および他ファイル (domain/agent.rs, project repo, worktree repo, migration 等) には変更なし
