## Important Decisions

- **デフォルト値の扱い**: `AgentCreateInput` の `role` / `adapter_type` / `status` は `Option<String>` とし、未指定時または空文字時に repo 層でデフォルト適用 (`""` / `"cli"` / `"idle"`)。SQL の DEFAULT 句に任せず Rust 側で埋めることで、戻り値 Agent が確定値を持つ (フロントエンドが再 fetch しなくて済む)
- **`active_worktree_id` の unset**: Phase 1 では `Option<String>` のみ対応。unset したい場合は後続フェーズで `Option<Option<String>>` + カスタム deserializer に移行する (チケット 1B05 Implementation Notes の方針)
- **`prompt_path` / `config_path` の扱い**: 現状 `update` では Some で上書き、None は既存値保持。これも unset 非対応 (同上)
- **project 存在チェックの方法**: `SELECT 1 FROM projects WHERE id = ? LIMIT 1` を `query_row(...).optional()?` で実装。`COUNT(*)` より軽量
- **ヘルパ `find_by_id`**: 1B04 と同様に private 関数として repo 内に閉じて実装。commands 層が必要になれば public 化を検討
- **テスト用 project 作成**: `create_project(conn, slug)` ヘルパを定義し、`project_repo::create` 経由で実 DB に project を 1 件用意してから agent を作成する (本番経路と同じパスを通すことで FK 整合性も暗黙に検証)
