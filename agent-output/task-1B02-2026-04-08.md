## 実装ポイント

- `include_str!("migrations/001_init.sql")` のパスは `migration.rs` からの相対 (`src/db/migration.rs` → `src/db/migrations/001_init.sql`)
- 空 SQL でも `execute_batch("")` は no-op として成功するため、テーブル本体が無くても
  `schema_migrations` への記録は正しく行われる
- トランザクション境界は `conn.transaction()?` → `tx.commit()?`。失敗時は Drop で自動 rollback
- `BTreeSet<i64>` を使うことで、将来 `MIGRATIONS` が増えても `contains` が低コスト
- 現状 `MIGRATIONS` は 1 要素のみ (`(1, "init", ...)`) だが、後続で要素を追加するだけで
  同じ runner で適用可能
